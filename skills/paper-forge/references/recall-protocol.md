# Recall Protocol

Recall tests whether the user can explain the paper independently.

Rules:

1. Ask one question at a time.
2. Wait for the user's answer.
3. Do not reveal the complete answer automatically.
4. Identify missing causal links, evidence boundaries, and assumptions.
5. Ask the user to improve the answer.
6. Reveal only if the user explicitly asks with phrases such as `I do not know`, `Tell me the answer`, `Reveal`, `告诉我答案`, or `这题我不会`.
7. Append the exchange to `learning/08_recall_log.md`.

Good recall questions ask about:

- The true problem.
- Why prior methods were insufficient.
- The key intervention.
- The causal mechanism.
- Which evidence supports which claim.
- What remains unproven.
- Whether anything transfers to the user's research direction.
