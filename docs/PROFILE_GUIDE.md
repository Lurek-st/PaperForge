# Profile 指南 / Profile Guide

## 中文

Profile 是一个 Markdown 文件，带少量 YAML frontmatter。

字段说明：

- `profile_version`：Profile 模板版本，当前为 `1`。
- `default_output_language`：默认输出语言，可为 `auto`、`English`、`Chinese` 或 `bilingual`。
- `Research Identity`：你的研究身份、背景和长期方向。
- `Priority Questions`：读论文时最关心的问题，例如能否部署、延迟、可靠性、数据泄漏风险。
- `Preferred Application Contexts`：你最关心的应用环境，例如工厂、机器人工作站、质检、仓储物流。
- `Output Preferences`：分析语言和输出形式偏好。
- `Terminology Preferences`：术语解释偏好，例如希望用工程语言解释 AI 和机器人概念。
- `Low Priority Topics`：希望降低优先级的主题。
- `Optional Notes`：仅写与论文分析有关的补充信息。

工业具身智能示例可包括：工业具身智能、智能制造自动化、机器人、机器视觉、数字孪生；判断标准可包括实时性、安全可靠、Sim2Real、传感器噪声、执行器不确定性、PLC / MES / SCADA 集成和维护成本。

## English

The Profile is a Markdown file with minimal YAML frontmatter.

Field guide:

- `profile_version`: Profile template version, currently `1`.
- `default_output_language`: Default output language, such as `auto`, `English`, `Chinese`, or `bilingual`.
- `Research Identity`: Your research identity, background, and long-term directions.
- `Priority Questions`: What you care about when reading papers, such as deployability, latency, reliability, or data leakage risk.
- `Preferred Application Contexts`: Target environments such as factories, robotic workcells, quality inspection, or warehouse logistics.
- `Output Preferences`: Analysis language and output preferences.
- `Terminology Preferences`: How concepts should be explained, such as using engineering language for AI and robotics.
- `Low Priority Topics`: Topics that should receive lower priority.
- `Optional Notes`: Only include extra information relevant to paper analysis.

A practical industrial embodied intelligence Profile may include industrial embodied intelligence, intelligent manufacturing automation, robotics, machine vision, and digital twins; judging criteria may include real-time behavior, safety and reliability, Sim2Real, sensor noise, actuator uncertainty, PLC / MES / SCADA integration, and maintenance cost.
